## 🔥 Pilot Purpose

The **AI Dependency Boundary Assurance Pilot** gives a company a fast, board-facing way to determine whether one AI dependency is becoming **load-bearing** before **authority**, **retention**, **jurisdiction**, **reversibility**, **exit**, **receipt**, **replay**, and **consequence exposure** are proven.

This is not a generic AI governance review. It is a **dependency-boundary pilot**.

---

## 🎯 Pilot Target

One real or representative AI dependency, such as:

- **internal ChatGPT / Copilot usage**
- **AI customer support agent**
- **vendor model API**
- **AI claims-routing assistant**
- **AI sales automation**
- **AI legal / clinical / financial assistant**
- **AI agent with tool access**
- **AI memory or RAG system**
- **workflow automation using model output**

---

## 🧾 Pilot Deliverables

| Deliverable | Purpose |
|---|---|
| ⚡ **Executive Board Memo** | Board-ready summary of the dependency boundary |
| 🧠 **Dependency Boundary Grid** | Authority, retention, jurisdiction, consequence, exit, replay |
| 🚦 **Red / Yellow / Green Card** | Clear board classification |
| 🧾 **Decision Receipt** | Formal proof of review and outcome |
| 🔁 **Replay Scenario Map** | Tests whether the boundary decision holds under changed facts |
| ⏱️ **Revalidation Trigger List** | Events that force review to reopen |
| 🎤 **Executive Readout** | Leadership-ready explanation of risk and next action |

---

## ⚔️ Pilot Decision Outcomes

- ✅ **Admit** — dependency may proceed
- 🟡 **Narrow** — dependency may proceed only in reduced scope
- 🔺 **Escalate** — board, legal, compliance, cyber, or sovereign review required
- ⛔ **Refuse** — consequence boundary cannot be established
- 🛑 **Halt** — existing deployment must pause
- 🔁 **Revalidate** — prior approval is no longer sufficient

## 🚨 Demo Scenario

This folder uses a fictional company:

**Northstar HealthPay**

Dependency under review:

**AI claims-routing assistant using vendor LLM + workflow API + persistent memory store**

Finding:

**Emerging load-bearing dependency with regulated workflow influence, persistent memory, vendor reliance, unclear authority, and incomplete exit proof.**

Recommended outcome:

**Narrow + Escalate**
